# Text Replacement Issues - Root Cause Analysis & Fixes

## Problem Summary

When generating templates, some sections were missing `text_updates` entirely, causing them to display placeholder text instead of customized content. Specifically:

1. **Section 7 (How It Works)** - No text_updates generated
2. **Potentially other sections** - Silent failures not reported to user

## Root Cause Analysis

### Issue 1: Missing Allowed Selectors

**File:** `/public/js/section-allowed-selectors.json`

This JSON file maps section filenames to their editable selectors. It's generated by running:
```bash
node build/build-allowed-selectors.mjs
```

**Problem:** The file can become outdated when:
- New section HTML files are added to `/sections/`
- Existing sections are modified
- The build script hasn't been run recently

**Impact:** When `generateSectionText()` runs (line 997 in `templates-creation/index.html`), it calls `resolveAllowedSelectors()` which returns an empty object if the section isn't in the JSON file. The function then:
1. Logs a `console.warn` (which user never sees)
2. Returns `null`  
3. Section is silently skipped during text generation

### Issue 2: Silent Failure Handling

**File:** `/templates-creation/index.html` lines 1002-1005, 1152-1156

**Original Code:**
```javascript
if (Object.keys(allowedSelectors).length === 0) {
    console.warn(`No allowed selectors found for ${sectionFile}, skipping text generation`);
    return null;  // ❌ Silent failure!
}

// Later...
for (const { sectionId, textUpdates } of results) {
    if (textUpdates) {  // ❌ Silently skips null values
        template.text_updates[sectionId] = textUpdates;
    }
}
```

**Problem:** Errors are logged to console only, never shown to the user. The template generation appears successful even though sections are missing text.

## Fixes Applied

### Fix 1: Regenerate Allowed Selectors ✅

**Action:** Ran `node build/build-allowed-selectors.mjs`

**Result:**
- Processed 178 section files
- Generated selectors for all sections including `fp-theme-how-it-works`
- File updated: `/public/js/section-allowed-selectors.json`

**Verification:**
```bash
jq '.["fp-theme-how-it-works"]' public/js/section-allowed-selectors.json
```
Returns 10 selectors including h2[0], p[0], h3[0-3], etc.

### Fix 2: Improve Error Handling & User Feedback ✅

**File Modified:** `/templates-creation/index.html`

**Changes Made:**

1. **Better error messages when selectors missing** (line ~1002):
```javascript
if (Object.keys(allowedSelectors).length === 0) {
    const errorMsg = `No allowed selectors found for section ${sectionId} (${sectionFile}). This section will not have text replaced.`;
    console.error(errorMsg);
    showStatus(`⚠️ Warning: ${errorMsg}`, 'warning');  // ✅ Show to user!
    return {};  // Return empty object instead of null
}
```

2. **Show warnings for API errors** (line ~1054):
```javascript
if (!response.ok || !data.success) {
    console.error(`Error generating text for section ${sectionId}:`, data.error);
    showStatus(`⚠️ Error generating text for section ${sectionId}: ${data.error || 'Unknown error'}`, 'warning');  // ✅ Show to user!
    return null;
}
```

3. **Summary report after text generation** (line ~1148):
```javascript
const failedSections = [];
const emptySections = [];

for (const { sectionId, textUpdates } of results) {
    if (textUpdates) {
        if (Object.keys(textUpdates).length === 0) {
            emptySections.push(sectionId);
        }
        template.text_updates[sectionId] = textUpdates;
    } else {
        failedSections.push(sectionId);
    }
}

// Show warnings if any sections failed
if (failedSections.length > 0) {
    showStatus(`⚠️ Warning: ${failedSections.length} section(s) failed text generation: ${failedSections.join(', ')}`, 'warning');
}

if (emptySections.length > 0) {
    showStatus(`⚠️ Warning: ${emptySections.length} section(s) have no selectors. Run 'node build/build-allowed-selectors.mjs'`, 'warning');
}
```

### Fix 3: Updated Existing Template (Temporary) ✅

**File Modified:** `/templates/fit-youth-gym-hub2.json`

Added missing `text_updates` for section 7 so the existing template works immediately. This is a **temporary fix** - the real fix ensures future templates won't have this issue.

## Prevention & Best Practices

### For Developers

1. **Always regenerate selectors after adding/modifying sections:**
   ```bash
   node build/build-allowed-selectors.mjs
   ```

2. **Check for warnings during template generation** - The UI now shows warnings if sections fail

3. **Verify text_updates completeness:**
   ```javascript
   // After generating a template
   const missingSections = template.sections.filter(id => !template.text_updates[id]);
   if (missingSections.length > 0) {
       console.error('Missing text_updates for sections:', missingSections);
   }
   ```

### Automated Solution (Recommended)

Add a check in the template generation workflow:

```javascript
// In templates-creation/index.html, before calling generateTemplate()
async function ensureAllowedSelectorsAreFresh() {
    const selectorsFile = '../public/js/section-allowed-selectors.json';
    const sectionsCount = (await glob('../sections/*.html')).length;
    
    const response = await fetch(selectorsFile);
    const selectors = await response.json();
    const selectorsCount = Object.keys(selectors).length;
    
    if (selectorsCount < sectionsCount) {
        showStatus('⚠️ Allowed selectors file is outdated. Please run: node build/build-allowed-selectors.mjs', 'error');
        return false;
    }
    return true;
}
```

## Conclusion

The root cause was:

2. **Silent error handling** - Failures not reported to user
