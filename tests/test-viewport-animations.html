Section 1

Welcome to Viewport Animations Test

This is a test page for the viewport animations feature. Scroll down to see animations trigger as elements enter the viewport.

Section 2

Animation Trigger Point

Elements will animate when they are 200px from the bottom of the viewport (by default). You can adjust this offset using the slider in the control panel.

Section 3

Multiple Elements

Each type of element has a staggered animation delay for a cascading effect.

Headings animate first, followed by paragraphs, images, and buttons.

Section 4

Intersection Observer

The viewport animations use the Intersection Observer API for efficient detection of when elements enter the viewport.

Section 5

Performance Optimized

Elements are automatically unobserved after they animate, reducing overhead and improving performance.

Section 6

Final Section

Use the "Reset Animations" button to remove all animations and start over. This is useful for testing different offset values.
